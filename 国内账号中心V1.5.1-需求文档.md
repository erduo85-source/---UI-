# 需求背景

:::
需求背景：

需求目标：

完善账号体系：解决后续手机换绑后导致的登录异常问题，引导历史玩家修复

优化玩家体验：提供标准化的自助账号服务，有效解决用户的账号管理需求

**手机登录调整**

:::
通行证侧调整：

通行证帐号新增“登录手机”字段

手机登录/注册时的校验逻辑调整如下文流程图所示

手机登录时，输入/取号的手机号 优先匹配登录手机进行登录

若未匹配到登录手机，则检查是否有老帐号是用该手机号注册的（用手机号 匹配账号名）

若没有，则进入注册流程

若有，则检查该老帐号，是否仍在使用该安全手机号进行验证码登录

如果老帐号有绑定登录手机，说明该手机号已被释放，本次输入/取号的手机号进入注册流程

如果老帐号没有绑登录手机，说明该手机号仍被占用，则判断该账号的安全手机是否为本次取号的手机号

若该老帐号的安全手机  等于     本次取号的手机号： 本次登录成功，进入老帐号（这种“安全手机==账号名”的老帐号，会自动处理绑定登录手机）

若该老账号的安全手机 不等于 本次取号的手机号：本次登录流程结束，通行证报错提示“该手机号已绑定其他账号”

注册时，账号名不在使用手机号

对于使用账号名作为游戏角色标识的游戏（例如手杀、OL、一将等）：通行证自动填入账号名，账号名生成格式与三方注册一致；

对于不用账号名作为游戏角色标识的游戏（例如指间山海、三武等 ）：通行证的账号名默认为空，后续允许玩家自助设置一次账号名；
:::

**登录手机补全**

:::
最终目标：功能上线后，大部分手杀玩家无需额外操作，即可使用手机登录

实现方案：针对满足以下条件的手杀通行证账号，自动补全/绑定登录手机

[x] 手机账号：绑定安全手机==账号名

[x] 个性账号：绑定安全手机未注册过手机账号，且该手机只绑定了该账号

[x] 三方账号：绑定安全手机未注册过手机账号，且该手机只绑定了该账号

预期效果：可自动绑定登录手机的手机号共计**30,806,391，**占比**97.82%**。在功能上线后，**3,591,219**个原本无法使用一键登录的个性/三方账号 可直接使用一键登录（无需去账号中心手动绑定）

备注说明：手机账号自动补全登录手机，三方/个性账号仅手杀玩家自动补全

*   Q1：为什么手机账号 必须满足“绑定安全手机==账号名”，才能自动补全登录手机
    
*   A1：部分换绑过的手机账号，自动绑定登录手机会破坏玩家原有的手机验证码登录流程
    

举例账号0：账号名==130、安全手机==152

当前线上：账号0可以使用验证码登录（输入账号130，接收短信通过152）

导致结果：若为账号0自动绑定152为登录手机，短信发送到152，会破坏原有的玩家手机验证码登录流程

*   Q2：为什么一绑一的个性账号和三方账号，还需要“绑定安全手机未注册过手机账号”
    
*   A2：存在其他账号 使用该手机号作为手机号验证码登录的输入
    

举例账号1：账号名==sgs、安全手机==130

举例账号2：账号名==130、安全手机==152

当前线上：账号2可以使用验证码登录。输入账号130，接收短信通过152

导致结果：账号1不能自动绑定130为登录手机

`**step1-安全验证**`**：**当且仅当绑定了安全手机或安全邮箱时验证，均未绑定时则跳过安全验证

*   验证方式
    
    *   输入框标题：安全手机/安全邮箱
        
    *   数据排序：当账号同时绑定手机和邮箱时，++优先展示手机验证++，可通过下拉切换验证方式为邮箱验证

    *   输入框标题：验证码
        
    *   默认展示：占位符文案+发送按钮
        
    *   占位符文案：请输入验证码

    *   滑块验证：首次触发发送时不触发滑块验证，后续300S内再次请求时触发滑块验证

|  什么是游卡通行证？  |  游卡通行证是登录游卡旗下所有平台（包括游戏、社区、桌游圈等）的统一账号。  |

|  一张身份证可以认证多个账号吗？  |  可以。但如果认证信息为未成年人，所有使用该身份证认证的账号均会被纳入防沉迷范围： *   累计在线5小时以上，游戏收益降为0；      *   同一身份证下的不同账号，在线时间累计计算。       |
|  一个安全手机或安全邮箱可以绑定多个账号吗？  |  可以。 *   安全手机：注册时即绑定该手机，该手机号也可作为其他游卡账号的安全手机；      *   安全邮箱：同样支持绑定多个账号。       |

|  什么是验证期？  |  新绑定的手机或邮箱，在绑定成功后的24小时内（1天） 为验证期。验证期内，该联系方式不能用于找回密码。  |
