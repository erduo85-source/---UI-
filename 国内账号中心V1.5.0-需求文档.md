# 需求背景

:::
需求背景：

账号逻辑：当前线上一个手机号可绑定多个账号，换绑后造成了一系列登录问题（详见：[《国内账号中心V1.5.0-需求文档》](https://docs.xiaoshanim.com/i/nodes/gNKMlbrq3WXA1d9YNdbF4Xv5p6Ad9nL1?utm_scene=team_space&iframeQuery=anchorId%3Duu_mohza2tjpcvhwz4yw4i)）

网页服务：服务分散，缺乏统一性；流程冗余影响转化；技术框架陈旧，难以维护和拓展。

需求目标：

完善账号体系：解决后续手机换绑后导致的登录异常问题，引导历史玩家修复

优化玩家体验：提供标准化的自助账号服务，有效解决用户的账号管理需求
:::

**登入中转页：**

*   页面入口：
    
    *   官网跳转→ 官网点击会员，跳转访问登录入口页
        
    *   URL访问→ 输入账号中心地址，访问首页
        
    *   账号管理→账号登出后，自动跳转该页面
        

*   页面逻辑：
    
    *   未完成登录：访问该页面时自动拉起登录弹窗，未完成登录时无法直接访问账号管理页
        
    *   完成登录/访问时浏览器已保存有效cookie→ **自动登录时，直接进入账号管理页**（跳过当前页）
        
*   页面交互：
    
    *   主按钮：点击“立即登录”按钮，触发统一登录弹窗
        
    *   立即登录：点击后拉起登录弹窗
        
    *   账号管理：点击后拉起登录弹窗
        
    *   账号服务：点击后跳转安全中心，展示安全中心功能入口页
        
*   统一登录：
    
    *   **遇到问题：按钮点击跳转改为安****全中心首页**[https://account.yokaverse.com/security](https://account.yokaverse.com/security)
